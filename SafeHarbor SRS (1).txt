Project Overview
SafeHarbor is a high-fidelity disaster response and emergency coordination platform designed to orchestrate systematic incident reporting, resource logistics, and personnel deployment natively. The system provides structural entry points for managing disaster scenarios, survival asset stockpiles, and personnel account assignments while rigorously encapsulating business logic for resource conflict validation, spatial coordinate sanity, and immutable audit logs securely
The platform enforces role-based access control dynamically allocating distinct privileges for AGENCY_DIRECTOR, EMERGENCY_DISPATCHER, and FIELD_RESPONDER roles inherently. All interactions strictly utilize a stateless filter topology that executes the standard HS256 JWT algorithm implicitly signing tokens to prevent unauthorized record modifications and natively restricted administrative operations internally safely.

Entity Models
PersonnelAccount Entity (Table: personnel_accounts)
Fields: id (Long, Primary Key), username (String), passwordHash (String), fullName (String), role (String), contactNumber (String), assignedRegion (String), isActive (boolean)
Validation: username @Column(unique=true, nullable=false), passwordHash @Column(name="password_hash", nullable=false), fullName @Column(name="full_name", nullable=false), role @Column(nullable=false).
Note: Role values are AGENCY_DIRECTOR, EMERGENCY_DISPATCHER, FIELD_RESPONDER. Default isActive=true.

DisasterIncident Entity (Table: disaster_incidents)
Fields: id (Long, Primary Key), title (String), description (String), incidentType (String), severityLevel (String), latitude (Double), longitude (Double), status (String), reportedAt (LocalDateTime), assignedResponder (ManyToOne -> PersonnelAccount)
Validation: title @Column(nullable=false), description @Column(columnDefinition="TEXT", nullable=false), incidentType @Column(name="incident_type", nullable=false), severityLevel @Column(name="severity_level", nullable=false), latitude @Column(nullable=false), longitude @Column(nullable=false), status @Column(nullable=false).

Note: Status values are REPORTED, ASSIGNED, RESOLVED, CANCELLED. reportedAt is set automatically on creation.

SupplyInventory Entity (Table: supply_inventories)
Fields: id (Long, Primary Key), itemName (String), category (String), availableQuantity (Integer), reservedQuantity (Integer), criticalThreshold (Integer), unit (String)
Validation: itemName @Column(name="item_name", unique=true, nullable=false), category @Column(nullable=false), unit @Column(nullable=false).
Note: Default availableQuantity=0, reservedQuantity=0, criticalThreshold=100.

ReliefShelter Entity (Table: relief_shelters)
Fields: id (Long, Primary Key), shelterName (String), locationAddress (String), capacity (Integer), currentOccupancy (Integer), managerName (String), isActive (boolean)
Validation: shelterName @Column(name="shelter_name", nullable=false), locationAddress @Column(name="location_address", nullable=false), capacity @Column(nullable=false).
Note: Default currentOccupancy=0, isActive=true.

ResourceDispatch Entity (Table: resource_dispatches)
Fields: id (Long, Primary Key), targetIncident (ManyToOne -> DisasterIncident), inventory (ManyToOne -> SupplyInventory), dispatchedQuantity (Integer), dispatchStatus (String), initiatedAt (LocalDateTime)
Validation: targetIncident @JoinColumn(name="target_incident_id", nullable=false), inventory @JoinColumn(name="inventory_id", nullable=false), dispatchedQuantity @Column(name="dispatched_quantity", nullable=false), dispatchStatus @Column(name="dispatch_status", nullable=false).
Note: DispatchStatus values are PENDING_APPROVAL, IN_TRANSIT, DELIVERED, CANCELLED.

Controller Layer
Controllers manage incoming external requests inherently leveraging @RestController for JSON serialization.

1.AuthController (/api/auth)

Success: Registers personnel account and returns response DTO.
POST /api/auth/login: (PUBLIC) 200 OK, returns ResponseEntity<AuthResponseDto>.
Success: Returns AuthResponseDto containing token (String), username (String), and role (String).
Failure: 401 Unauthorized if credentials are invalid.
GET /api/auth/personnel: (DIRECTOR) @PreAuthorize("hasAuthority('AGENCY_DIRECTOR')") 200 OK, returns ResponseEntity<Page<PersonnelAccountResponseDto>>.

Success: Logs a new incident and returns the saved record.
GET /api/incidents: (AUTHENTICATED) 200 OK, returns ResponseEntity<Page<DisasterIncidentResponseDto>>.
Success: Returns all incidents (paginated).
PATCH /api/incidents/{id}/status: (AUTHENTICATED) 200 OK, returns ResponseEntity<Void>.
Accepts @RequestParam String status.
Success: Transitions incident lifecycle state (e.g., to RESOLVED).

3.InventoryController (/api/inventory)
POST /api/inventory: (DIRECTOR/DISPATCHER) @PreAuthorize("hasAnyAuthority('AGENCY_DIRECTOR', 'EMERGENCY_DISPATCHER')") 201 Created, returns ResponseEntity<SupplyInventoryResponseDto>.

Security Implementation
JWT signing algorithm: HS256
Security utility class name: JwtService (located in com.example.demo.security.JwtService)
Exact method names in JwtService.java: generateToken (accepts UserDetails parameter, returns String), isTokenValid (accepts String token and UserDetails userDetails parameters, returns boolean), extractUsername (accepts String token, parses JWT subject and returns username String)
JwtService secretKey field must be declared private (injected via @Value("${safeharbor.jwt.secret}")).
SecurityConfig.java: Annotated with @Configuration, @EnableWebSecurity, and @EnableMethodSecurity.
Bean Declarations: Must declare a @Bean method named passwordEncoder returning BCryptPasswordEncoder and a @Bean method named securityFilterChain returning SecurityFilterChain.
CORS Policy: Must declare a @Bean method named corsConfigurationSource containing CORS configuration. Must allow origins http://localhost:3000 and http://localhost:5173. Allowed methods: GET, POST, PUT, PATCH, DELETE, OPTIONS. Allowed headers: Authorization, Content-Type, X-Requested-With, Accept.
Path Authorization:
/api/auth/** ? permitAll() (PUBLIC)
/api/** ? authenticated() (JWT required, method-level @PreAuthorize handles role enforcement)
Session Management: Explicitly configured as STATELESS (SessionCreationPolicy.STATELESS).
JwtAuthenticationFilter extends OncePerRequestFilter and is added before UsernamePasswordAuthenticationFilter.

Exception Handling
Required Package: com.example.demo.exception The central handler class is GlobalExceptionHandler. It must be annotated with @RestControllerAdvice. All error responses use a Map<String, Object> body containing keys timestamp, status, and message.
ResourceNotFoundException: Thrown when a PersonnelAccount or DisasterIncident cannot be found by ID or username. The @ExceptionHandler method must return ResponseEntity with HTTP 404 NOT_FOUND and a Map body containing the error message.
BusinessValidationException: Thrown when operational constraints are violated (e.g., inventory shortage or invalid coordinate bounds). The @ExceptionHandler method must return ResponseEntity with HTTP 400 BAD_REQUEST and a Map body containing the error message.
RuntimeException (generic): Catches all unhandled runtime errors including auth failures and internal orchestration exceptions. The @ExceptionHandler method must return ResponseEntity with HTTP 500 INTERNAL_SERVER_ERROR and a Map body containing key message set to the exception details.

store/slices/authSlice.js
Purpose: Maintains authentication session state securely.
Implementation Requirements:
- Initial State: { user: null, token: null, loading: false, error: null }. Initializes user and token from localStorage if present.
- Action clearAuthError: Resets error to null.
- Action logout: Removes 'safeharbor_user' and 'safeharbor_jwt_token' from localStorage, sets user: null, token: null, loading: false, error: null in Redux state.
- Async Action login (type prefix 'auth/login'): pending sets loading=true; fulfilled sets loading=false, token=payload.accessToken, user=payload (containing username, role, fullName); rejected sets loading=false, error=payload.
- After successful login, the auth state must have token and user populated with the response data from backend.
- logout action must reset the entire auth state to initial values (user: null, token: null, loading: false, error: null).
store/slices/incidentSlice.js
Purpose: Manages disaster incident records and UI filter state.
Implementation Requirements:
Initial State: { items: [], loading: false, error: null, currentIncident: null, filterStatus: 'ALL' }.
Action selectIncident: Sets currentIncident to payload.
Action setFilterStatus: Sets filterStatus to payload.
Async Action fetchIncidents (type prefix 'incidents/fetchAll'): pending sets loading=true; fulfilled sets loading=false, items=payload.
Async Action reportIncident: fulfilled pushes the new incident into the items array.
store/slices/inventorySlice.js
Purpose: Manages survival asset stockpile records.
Implementation Requirements:
Initial State: { items: [], loading: false, error: null, shortages: [] }.
Async Action fetchInventory: fulfilled sets items=payload and filters shortages based on criticalThreshold.
Async Action updateStock: fulfilled updates the quantity of the specific item in the items array.
store/slices/dispatchSlice.js
Purpose: Manages resource dispatch orchestration records.
Implementation Requirements:
Initial State: { items: [], loading: false, error: null }.
Async Action fetchDispatches: fulfilled sets items=payload.
Async Action requestDispatch: fulfilled adds the new dispatch record to items.
store/slices/shelterSlice.js
Purpose: Manages relief shelter records and occupancy state.
Implementation Requirements:
Initial State: { items: [], loading: false, error: null }.
Async Action fetchShelters: fulfilled sets items=payload.
Async Action updateOccupancy: fulfilled updates the currentOccupancy for the specific shelter in the items array.
store/slices/personnelSlice.js
Purpose: Manages personnel directory records for administrative views.
Implementation Requirements:
Initial State: { items: [], loading: false, error: null }.
Async Action fetchPersonnel: fulfilled sets items=payload.
Async Action deactivatePersonnel: fulfilled updates the isActive flag for the specific account in the items array.
